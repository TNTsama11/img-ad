# 模型发现与选择设计

## 目标

解决三个直接影响使用的问题：创作页右上角两个文字按钮含义不清、当前模型选择入口不明显，以及新增供应方时只能手工录入模型。

## 交互设计

### 创作页

- 顶栏只保留页面标题，移除“供应方”和“模型”两个独立文字按钮。
- 在标题下方增加全宽“当前模型”选择栏，主文案显示模型名称，副文案显示供应方名称；未配置时显示“选择模型”和明确的引导状态。
- 点击选择栏打开底部抽屉。模型按供应方分组，每一项同时展示供应方与模型名称，当前项使用单选状态标识。
- 选择模型时同时切换供应方和模型，避免出现“供应方已切换但模型尚未选择”的中间状态。

### 供应方编辑页

- 保留名称、Base URL、API Key 与手动保存能力。
- 增加“获取模型”按钮，仅在用户点击后请求 `GET {baseUrl}/models`，不自动产生网络请求。
- 获取成功后显示搜索框和可多选模型列表。已存在的同名模型默认选中，再次导入使用确定性 ID 更新，不创建重复项。
- 主操作根据选择状态显示“保存并导入模型（N）”；未获取或未选择模型时仍可只保存供应方。
- 获取失败时显示可读错误，保留重试和手动新增模型入口。

## 数据与协议

- 新增单一职责端口 `FetchProviderModels`，输入供应方、可选 API Key 覆盖值，输出模型 ID 列表。
- 远程实现复用 `RemoteUrlPolicy`、`ValidatingDns` 和 OkHttp 安全配置，发送 Bearer Token，并关闭重定向。
- 解析 OpenAI 兼容响应 `{"data":[{"id":"gpt-image-1"}]}`；忽略空 ID，按服务端顺序去重。
- 编辑现有供应方且 API Key 留空时，从安全存储读取已有密钥；新增供应方必须填写 API Key 才能获取模型。
- 导入模型默认仅启用文生图能力。编辑、蒙版、多图、尺寸和质量能力无法由标准 `/models` 响应可靠推断，继续通过现有模型编辑页配置。

## 状态与错误

- `SettingsUiState` 保存获取中状态、候选模型、已选 ID、搜索词和获取错误。
- 每次 Base URL 或 API Key 改变后，由编辑页清空旧的发现结果，避免把上一组凭据获取的模型导入新供应方。
- HTTP 非成功状态显示状态码；空列表、响应格式错误、缺少密钥和网络错误分别给出明确提示。
- 保存供应方成功后再逐个保存所选模型，满足 Room 外键约束；任一步失败均留在当前页面并展示错误。

## 测试策略

- JVM：MockWebServer 验证 `/models` 路径、Bearer 头、解析去重、HTTP/格式错误与密钥回退。
- ViewModel：验证显式获取、选择/搜索、保存并导入、确定性 ID、失败状态。
- Compose：验证创作页模型入口及联动选择；供应方编辑页获取、搜索、多选和导入操作可发现。
- 完整验证：JVM 单元测试、Android 仪器测试、lint、Debug 构建，并在现有 `imgad_api35` 模拟器安装后人工检查主要流程。
