# 会话中心与后台生成设计

## 目标

将现有历史页升级为统一的会话管理中心，并将生成任务从当前创作页面生命周期中解耦。用户可以在不同会话并发生成图片，同时查看设置、其他会话或新建创作，不会取消或覆盖正在运行的任务。

本设计同时覆盖：MIT 授权、历史提示词复制、生成等待指示器、会话运行标识，以及此前确认的“历史记录显示最近使用模型”设计。

## 任务协调架构

应用新增 `GenerationCoordinator`，由 `App` 持有，生命周期与应用进程一致。协调器使用 `SupervisorJob` 隔离不同会话任务，并按 `sessionId` 保存运行中的 Job。

- 不同会话可以并发生成。
- 同一会话最多存在一个运行任务，重复提交会被拒绝。
- 取消操作只取消指定会话的任务。
- 单个会话失败不取消其他会话。
- 切换设置、会话列表、其他会话或新建创作不取消任务。
- 运行中的会话不可删除，必须先取消生成。

`CreateViewModel` 不再拥有网络 Job，也不在 `loadSession`、`openSession` 或 `resetToNewSession` 中取消生成。它只管理当前可见会话、编辑器草稿、消息订阅和页面事件，并组合协调器的运行状态得出当前页面是否正在生成。

编辑器草稿按会话 ID 保存在 `CreateViewModel` 的进程内草稿映射中，新建创作使用独立草稿键。切换会话时保存当前草稿并恢复目标会话草稿，避免提示词和参考图串到其他会话。草稿不写入数据库，应用进程终止后不保证恢复。

应用进程被系统终止时不承诺继续联网。下次启动沿用现有恢复逻辑，将数据库中残留的 `RUNNING` 消息标记为中断失败。

## 数据与状态来源

生成用例继续在请求开始时写入 `RUNNING` 用户消息，并在成功、失败或取消时更新数据库。数据库是会话历史和最终任务状态的权威来源。

协调器只保存进程内 Job 和即时运行会话集合，不新增任务表，不升级数据库版本。它暴露不可变的运行会话 ID 集合，供创作页和会话页订阅。

会话列表使用只读投影读取每个会话最后一条带请求快照的消息，并从结构化 JSON 中解析最近使用的模型。模型快照规则沿用 `2026-07-15-history-model-snapshot-design.md`：

- 多模型会话显示最近一次请求的模型；
- 失败或取消的最近请求同样计入；
- 快照缺失、损坏或不含模型时显示“模型未知”；
- 不关联当前模型配置，历史值不随模型重命名或删除而变化。

## 会话管理界面

底部导航“历史”改名为“会话”，页面标题同步改为“会话”。该页面同时承担历史浏览和会话管理。

每张会话卡片显示：

- 会话标题；
- 最近使用模型；
- 运行状态；
- 重命名和删除菜单。

正在生成的会话显示绿色状态点和“生成中”。状态点只代表协调器中存在该会话的活动 Job。点击卡片主要区域进入会话，不再要求精确点击标题。

运行中的会话点击删除时不打开删除确认框，而是显示“请先取消生成”。非运行会话保持现有重命名和删除流程。

## 创作界面等待状态

提交请求后，数据库写入的 `RUNNING` 消息立即出现在消息列表。运行消息显示转圈指示器和“生成中”，使等待状态位于结果将出现的位置。

编辑器区域在当前会话运行时显示全宽“取消”按钮，并禁止该会话重复提交。切换页面再返回时，等待指示器和取消按钮根据协调器及消息流恢复。

其他会话正在运行不会禁用当前会话编辑器。用户可以在新会话或其他空闲会话提交任务，实现跨会话并发。

## 提示词复制

每条用户提示词消息提供复制图标按钮，并支持长按提示词正文复制。图标按钮使用 Compose Material 图标及可访问描述“复制提示词”。

两种交互调用同一 `onCopyPrompt` 回调，将完整提示词写入系统剪贴板。复制成功后使用全局 Snackbar 显示“提示词已复制”。空提示词不显示复制操作。

## 异常与一致性

- 协调器启动任务前原子检查同一会话是否已有 Job。
- Job 完成、失败或取消后必须在 `finally` 中移除运行标识。
- 生成用例继续负责把失败和取消落库；协调器不复制错误持久化逻辑。
- 页面不根据当前可见会话推断其他会话状态，只订阅协调器的运行集合。
- 运行中删除在 UI 和 ViewModel 两层拒绝，避免竞态导致文件被删除后网络结果仍写入。
- 应用启动恢复只处理数据库残留状态，不尝试恢复已经丢失的进程内 Job。

## 测试

### 协调器单元测试

- 不同会话可以并发运行；
- 同一会话拒绝重复任务；
- 取消指定会话不影响其他会话；
- 失败隔离；
- 任务终止后移除运行状态。

### ViewModel 与仓库测试

- 切换设置、其他会话和新建创作不取消任务；
- 当前会话运行时禁止重复提交，其他会话仍可提交；
- 运行中会话禁止删除；
- 最近模型投影、损坏快照和旧记录降级。

### Compose 与验收测试

- 运行消息显示转圈指示器；
- 离开并返回会话后等待状态仍存在；
- 会话卡片显示绿色状态点、“生成中”和最近模型；
- 卡片主要区域可进入会话；
- 复制图标与长按均复制完整提示词并显示 Snackbar；
- 多会话并发时分别完成、失败和取消互不影响。

最后运行完整 JVM 测试、Android 仪器测试、lint、Debug 构建，并在 API 35 模拟器手动验证至少两个并发会话的页面切换和状态恢复。
