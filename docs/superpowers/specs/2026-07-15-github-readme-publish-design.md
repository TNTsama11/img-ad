# GitHub 双语 README 与公开发布设计

## 目标

将当前 Android AI 图片客户端初始化为独立 Git 仓库，创建中英文项目说明，并发布到 GitHub 公开仓库 `img-ad`。公开内容仅包含源码、构建配置、Room schema、项目文档和经过筛选的界面截图。

## README 结构

- `README.md` 为中文主页，顶部提供英文版链接。
- `README_EN.md` 为内容完整的英文副本，顶部提供中文版链接。
- 两份文档保持相同章节顺序：项目简介、主要功能、界面截图、系统要求、构建安装、供应方配置、使用流程、测试、隐私安全、当前限制。
- 不宣称存在正式签名 Release、应用商店版本或开源许可证。
- 不写入真实 API 地址、API Key、用户本地路径或其他测试凭据。

## 截图

从已完成模拟器验证的截图中选取创作详情、猫咪全屏预览和历史页，复制到 `docs/images/`。README 使用仓库相对路径引用图片，避免依赖本地构建目录。

截图只展示应用界面和生成结果，不包含 API Key、供应方凭据、设备账号或通知内容。

## Git 忽略规则

根目录新增 Android/Kotlin 项目 `.gitignore`，至少排除：

- `.gradle/`、`.kotlin/`、所有 `build/` 目录；
- `local.properties`、IDE 本地文件、日志和临时文件；
- APK、AAB、映射文件和模拟器测试产物；
- keystore、JKS、PEM、私钥和常见环境密钥文件。

## 发布前检查

- 检查待提交文件清单和总体积。
- 扫描 OpenAI 风格密钥、私钥头、常见密钥赋值和签名文件。
- 运行现有 JVM 测试、Android 仪器测试、lint 和 Debug 构建；若发布前代码未变化，可引用同一工作区内最新完整结果，但 README 与忽略规则仍需单独检查。
- 检查 README 中英文链接、截图路径和构建命令有效。

## GitHub 发布

- 本地默认分支为 `main`。
- 首次提交包含确认范围内的全部源码、文档和必要资源。
- 使用 GitHub CLI 登录当前用户，创建公开仓库 `img-ad`，设置 `origin` 并直接推送 `main`。
- 新仓库首次发布不创建 Pull Request。
- 若同名仓库已存在、认证用户不明确或敏感信息扫描命中，则停止并要求确认，不覆盖远程内容。

## 安全边界

仓库公开意味着推送内容可被任何人读取和复制。发布流程不上传应用私有数据库、Keystore 数据、API Key、生成缓存、APK 或本地工具链。GitHub 登录令牌只由 GitHub CLI 凭据存储管理，不写入项目目录。
