# 历史记录模型快照设计

## 目标

历史列表为每个会话展示最近一次生成请求实际使用的模型。展示值来自生成当时保存的请求快照，不随当前供应方或模型配置的重命名、禁用、删除而变化。

## 展示规则

- 历史卡片在会话标题下显示 `模型：<模型技术名>`。
- 会话使用过多个模型时，显示最后一条带请求快照的消息中的模型。
- 最近一次失败或取消的请求同样视为最近使用的模型。
- 快照缺失、格式损坏或不含有效模型时显示 `模型未知`。

## 数据设计

现有 `messages.requestSnapshotJson` 已不可变地保存 `providerId` 和模型技术名 `model`，因此不向 `sessions` 表冗余写入模型，也不升级数据库版本。

`SessionDao` 新增历史列表专用只读投影查询。投影包含完整会话实体和该会话最后一条非空请求快照。最后一条消息按 `createdAt DESC, id DESC` 确定，确保时间相同时顺序稳定。现有会话查询继续服务归档等调用方，不改变其契约。

领域模型 `Session` 增加可空只读摘要字段 `latestModel`，默认值为 `null`，以保持现有调用方源码兼容。`SessionRepository` 将投影映射到该字段，不新增另一套历史专用领域类型。快照通过结构化 JSON API 解析；任何解析异常只影响模型标签，不能中断整个历史列表。

## 界面设计

历史卡片保留标题、打开会话、更多菜单、重命名和删除行为。在标题下增加一行较弱层级的模型标签，不增加新的操作控件。

## 兼容性

- 不需要 Room 数据库迁移。
- 归档格式不变，请求快照已经随消息导出和导入。
- 旧会话可以从已有快照直接显示模型；无有效快照的旧会话显示 `模型未知`。
- 不读取 API Key，也不把密钥写入历史摘要。

## 测试

- DAO 仪器测试验证多条消息时选择最近的有效请求快照，并验证同时间戳时按消息 ID 稳定排序。
- Repository 单元测试验证模型解析、缺失快照和损坏快照降级为未知。
- Compose 仪器测试验证历史卡片显示 `模型：gpt-image-2` 和 `模型未知`。
- 运行完整 JVM 测试、Android 仪器测试、lint 和 Debug 构建。
